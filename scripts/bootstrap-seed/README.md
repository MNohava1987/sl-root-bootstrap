# Spacelift Zero-Click Bootstrap Seed

This directory contains the foundational Terraform logic required to initialize a fresh Spacelift account with its first "Seed" identity. This process bypasses the UI and establishes the management plane entirely through code.

## Prerequisites

1. **GitHub Integration**: You must have the Spacelift GitHub App installed on your account.
2. **VCS Integration ID**: Obtain the ID of your GitHub integration from the Spacelift UI (required for the `sl-root-bootstrap` stack).
3. **Admin API Key**: Create an API key with Admin permissions in the Spacelift UI once to obtain the credentials.

## Authentication

This process uses Environment Variables for authentication. Ensure the following are set in your terminal:

```bash
export SPACELIFT_API_KEY_ENDPOINT="https://your-account.spacelift.io"
export SPACELIFT_API_KEY_ID="01K..."
export SPACELIFT_API_KEY_SECRET="..."
```

## How to Seed

1. Navigate to this directory:
   ```bash
   cd scripts/bootstrap-seed
   ```
2. Initialize and Apply:
   ```bash
   export TF_VAR_vcs_integration_id="your-vcs-id"
   terraform init
   terraform apply -auto-approve
   ```

## What this creates

- The foundational **`sl-root-bootstrap`** stack in the `root` space.
- An explicit **`Space Admin`** RBAC role attachment for the stack.
- Modern security hardening (Protection from deletion, local previews enabled).

## Handoff

Once this local run is successful, the `sl-root-bootstrap` stack will take over the management of the account. You can then delete any local state files generated by this process.
